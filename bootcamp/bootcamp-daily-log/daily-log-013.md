## 2교시 (10:00 ~ 11:00)

**주제:** GitHub Project 생성 및 관리

- Project, Issue, Milestone 만들기

**브랜치 규칙 정리**

- 메인 브랜치(main)에는 직접 커밋하지 않기
- 기능 개발 전, main 브랜치를 기준으로 새로운 브랜치 생성
- 브랜치 이름은 `feature/기능이름` 형식으로 지정  
  　 → 예시: `feat/login`, `feat/cart`

- feature 브랜치는 한 명만 커밋
- 기능 개발 완료 후 main 브랜치와 병합

> Branch ↔ Branch 전환 명령어: `checkout`
